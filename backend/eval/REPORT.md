# PaperPal eval report

_Generated 2026-05-13 — v0.9.0 retrieval ablation_

## Dataset

- **26 questions across 4 papers.** Up from the v0.4.0 baseline of 12 questions on a single paper.
- **Papers:** *Attention is all you need* (Vaswani 2017, the original eval), *BERT* (Devlin 2018), *LoRA* (Hu 2021), *CLIP* (Radford 2021).
- **Curation honesty:** the 12 *Attention* questions are hand-written with hand-verified gold pages. The 14 new questions (BERT/LoRA/CLIP, ~5 each) were **generated by the local Llama 3.1 8B model from chunks sampled across each paper's pages**, with a light pass to remove citation-flavored hallucinations (4 questions pruned). LLM-generated questions are systematically easier and more lexically aligned with the source than human ones, which biases the eval. The next eval expansion should be hand-curated.
- Each question is scoped to its own paper's `paper_id` so retrieval doesn't have to choose between papers — this isolates the retrieval-quality signal from the cross-document-routing signal.

## Retrieval ablation (citation accuracy on the full 26-question set)

Four configurations of the retrieval pipeline:

```
            ┌───────────────────────────────────────────────────────┐
            │ dense (top-24)                                        │
config A:   │   └──> top-K to LLM                                   │  ── dense only
            │ dense (top-24)                                        │
config B:   │   └──> cross-encoder rerank ──> top-K to LLM          │  ── + reranker
            │ dense + BM25 (top-24 each)                            │
config C:   │   └──> RRF fuse ──> top-K to LLM                      │  ── + hybrid (no rerank)
            │ dense + BM25 (top-24 each)                            │
config D:   │   └──> RRF fuse ──> cross-encoder rerank ──> top-K    │  ── + both
            └───────────────────────────────────────────────────────┘
```

| Config | Citation precision | Citation recall | Citation **F1** | Δ vs. dense |
|---|---:|---:|---:|---:|
| Dense only | 0.346 | 0.385 | 0.359 | — |
| **+ Cross-encoder reranker** | **0.397** | **0.442** | **0.410** | **+0.051 (+14% rel.)** |
| + BM25 hybrid (no rerank) | 0.327 | 0.365 | 0.340 | −0.019 |
| + BM25 hybrid + reranker | 0.385 | 0.442 | 0.404 | +0.045 |

**Headline:** the cross-encoder reranker is the single most valuable retrieval stage, giving +14% relative F1 over the dense baseline. Hybrid retrieval (BM25 + dense, fused via Reciprocal Rank Fusion with k=60) **does not help on this dataset** — alone it underperforms dense, and combined with the reranker it slightly underperforms the reranker alone. Shipped default is therefore `RERANKER_MODEL=cross-encoder/...` with `HYBRID_RETRIEVAL=false`.

## Per-paper breakdown (F1)

| Paper | n | Dense | + Reranker | + Hybrid | + Hybrid + Reranker |
|---|---:|---:|---:|---:|---:|
| Attention (hand-curated) | 12 | 0.528 | **0.597** | 0.569 | 0.569 |
| BERT (LLM-gen) | 5 | 0.200 | 0.300 | 0.200 | **0.333** |
| LoRA (LLM-gen) | 5 | 0.200 | 0.200 | **0.000** | 0.200 |
| CLIP (LLM-gen) | 4 | 0.250 | 0.250 | 0.250 | 0.250 |

A few honest observations:

- **The hand-curated Attention questions are by far the easiest** to score well on (F1 0.53–0.60 vs. 0.20–0.33 for LLM-generated). This is partly an eval-quality artifact (the LLM-generated questions are sometimes ambiguous and may have multiple plausible answer pages) and partly real signal (the Transformer paper has very concrete factoid content that's friendly to RAG).
- **Hybrid retrieval catastrophically hurts LoRA** (F1 0.20 → 0.00). Inspecting LoRA results, BM25 surfaces chunks containing exact term matches that aren't on the gold page (e.g. "LoRA" appears on most pages, drowning the rank signal). RRF then pulls these into the candidate pool and they displace the genuinely relevant chunks. The reranker recovers some of this loss but not all of it.
- **Hybrid retrieval helps BERT** (best F1 0.333 with both). BERT mixes lexical terms ("MLM", "WordPiece") with semantic content; BM25's lexical signal pairs well with the reranker's joint scoring.
- **CLIP is flat at F1=0.25 across all configs.** All retrieval strategies retrieve the same kinds of chunks; the bottleneck here is somewhere else (probably the LLM picking citations imprecisely from a long paper).

## What this validates

- **v0.7.0 (reranker) was correct.** Confirmed on a 2× larger, 4× more diverse eval. Reranker is the canonical default.
- **v0.8.0 (hybrid retrieval) was correctly shipped default-OFF.** The single-paper micro-eval suggested hybrid hurts; the multi-paper scaled eval confirms the same pattern. The architecture is in the repo, the flag is documented, and it's now demonstrably the right default to leave off — until either a different fusion strategy (weighted instead of RRF) or a use case (cross-document retrieval where BM25's strength actually applies) appears.

## What this does NOT validate

- **Generalization beyond scientific papers** — the eval is still entirely papers.
- **The "right" hyperparameter choices.** We swap the reranker model (or remove it) but everything else (chunk size, embedding model, top-k, prompt) is held fixed.
- **Human-judgment correlation.** Citation accuracy is a proxy: it scores whether the model cited the correct page, not whether the *answer text* is correct. A scaled human-Likert eval is the obvious next step.

## Historical comparison

For reference, the v0.4.0 RAG-vs-no-RAG comparison on the original 12-question Attention-only eval established that retrieval itself adds substantial value over zero-shot generation:

| Metric | No-RAG | RAG (v0.4.0) | Lift |
|---|---:|---:|---:|
| Citation precision | 0.000 | 0.500 | +0.500 |
| Citation recall | 0.000 | 0.625 | +0.625 |
| Citation F1 | 0.000 | 0.542 | +0.542 |

The v0.9.0 scaled-eval F1 of 0.410 is *lower* than the v0.4.0 single-paper F1 of 0.542. Most of the gap is the LLM-generated questions being harder than hand-curated ones, not a regression in retrieval quality (the Attention sub-score actually *improved* with the reranker, from 0.542 to 0.597).
